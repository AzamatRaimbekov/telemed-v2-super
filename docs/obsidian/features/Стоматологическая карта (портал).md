# Стоматологическая карта (портал пациента)

> Портал пациента: просмотр зубной карты (FDI-нотация), история лечения зубов, напоминание о профчистке.

---

## Обзор

Пациент видит свою зубную карту в интерактивном формате: 32 зуба, раскрашенные по статусу. Клик по зубу открывает историю лечения. Баннер вверху напоминает о профчистке, если прошло более 6 месяцев.

---

## Архитектура

### Модели данных

**Таблица:** `dental_charts`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | UUID | PK |
| `clinic_id` | UUID | Мультитенант |
| `patient_id` | UUID | FK -> patients (unique) |
| `teeth` | JSON | Карта зубов: `{"11": {"status": "healthy", ...}}` |
| `notes` | Text | Общие заметки |

**Таблица:** `tooth_treatments`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | UUID | PK |
| `clinic_id` | UUID | Мультитенант |
| `patient_id` | UUID | FK -> patients |
| `tooth_number` | Integer | Номер зуба (FDI: 11-48) |
| `doctor_id` | UUID | FK -> users |
| `procedure_name` | String(200) | Название процедуры |
| `diagnosis` | String(300) | Диагноз |
| `tooth_status_before` | String(20) | Статус до |
| `tooth_status_after` | String(20) | Статус после |
| `materials_used` | String(500) | Использованные материалы |
| `price` | Float | Стоимость |
| `notes` | Text | Примечания |

### Бэкенд

- **Модели:** `backend/app/models/dental_chart.py`
- **Маршруты:** `backend/app/api/v1/routes/dental_portal.py`

### Фронтенд

- **Страница:** `frontend/src/routes/portal/_portal/dental-chart.tsx`

## Статусы зубов

| Статус | Цвет | Описание |
|--------|------|----------|
| healthy | Белый | Здоровый зуб |
| caries | Красный | Кариес |
| filled | Синий | Пломба |
| crown | Золотой | Коронка |
| implant | Фиолетовый | Имплант |
| missing | Серый | Отсутствует |
| root_canal | Оранжевый | Каналы |
