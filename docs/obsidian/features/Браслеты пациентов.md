# Браслеты пациентов (Patient Wristband ID System)

> Система идентификационных браслетов для пациентов. Каждый пациент при поступлении получает уникальный браслет с кодом MC-XXXXXX, который можно сканировать через NFC, штрихкод или QR-код.

## Единый сервис идентификации (PatientIdentificationService)

Все способы идентификации пациента объединены в один сервис `backend/app/services/patient_identification.py`.

**Методы идентификации (в порядке приоритета):**
1. Браслет UID (`MC-XXXXXX`)
2. Штрихкод браслета
3. NFC метка браслета
4. UUID пациента
5. ИНН
6. Телефон

**Один эндпоинт для всего:**
```
GET /api/v1/qr/identify/{code}
```
Принимает любой код — автоматически определяет метод и возвращает полные данные пациента + информацию о браслете.

**Используется в:**
- QR-сканер (`/qr/scan/{id}`)
- Сканер браслетов (`/wristbands/scan/{code}`)
- Фронтенд: `identifyPatient(code)` из `features/qr/api.ts`

---

## Обзор

Браслетная система обеспечивает быструю и безошибочную идентификацию пациентов на всех этапах пребывания в клинике: при приёме лекарств, проведении процедур, лабораторных исследованиях и выписке.

### Формат идентификатора

- **UID:** `MC-XXXXXX` (6 символов, буквы A-Z + цифры 0-9)
- **Штрихкод:** `MCXXXXXX` (без дефиса, для Code128/EAN)
- **NFC:** привязка внешнего NFC-тега (опционально)
- **QR-код:** JSON с типом `medcore_wristband` и UID

### Статусы браслета

| Статус | Описание | Цвет |
|--------|----------|------|
| `active` | Активный, носится пациентом | Зелёный |
| `deactivated` | Деактивирован вручную | Серый |
| `lost` | Утерян, требуется замена | Красный |
| `discharged` | Снят при выписке | Синий |

## API Эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/v1/wristbands/issue` | Выдать браслет пациенту |
| GET | `/api/v1/wristbands/scan/{code}` | Сканировать браслет (UID/штрихкод/NFC) |
| GET | `/api/v1/wristbands/patient/{id}` | Активный браслет пациента |
| GET | `/api/v1/wristbands/patient/{id}/history` | История браслетов |
| POST | `/api/v1/wristbands/{id}/deactivate` | Деактивировать |
| POST | `/api/v1/wristbands/{id}/lost` | Отметить утерянным |
| GET | `/api/v1/wristbands/{id}/qr` | QR-код браслета (PNG) |
| GET | `/api/v1/wristbands/{id}/print` | Печатная этикетка (HTML) |

### Ответ на сканирование

При сканировании возвращаются:
- Данные браслета (UID, статус, дата выдачи)
- Полная информация о пациенте (ФИО, дата рождения, группа крови)
- Аллергии
- Назначение палаты
- Активные назначения лекарств

## Бизнес-логика

1. **Выдача:** При выдаче нового браслета все предыдущие активные браслеты автоматически деактивируются
2. **Уникальность:** UID генерируется с проверкой уникальности (до 10 попыток)
3. **Поиск:** Сканирование ищет по UID, штрихкоду и NFC-тегу последовательно
4. **Печать:** HTML-этикетка оптимизирована под термопринтер (50x25 мм)

- **Недавние сканирования** — список последних 10 сканирований
- **Действия** — печать, деактивация, утеря, переход к профилю

### Навигация
