# Протокол операций

Полный цикл ведения хирургических операций: планирование, проведение, протоколирование, печать.

## Возможности

- Планирование операции с датой, операционной, бригадой
- Запуск операции (фиксация start_time)
- Завершение (автоматический расчёт длительности)
- Отмена / перенос
- Печатный протокол в HTML (формат A4)
- Расписание операций на N дней вперёд

## Поля протокола

| Поле | Описание |
|------|----------|
| `surgery_name` | Название операции |
| `surgery_code` | Код по ОМС |
| `diagnosis` | Диагноз |
| `anesthesia_type` | Вид анестезии |
| `surgeon_id` | Хирург |
| `assistant_ids` | Ассистенты (JSON массив) |
| `anesthesiologist_id` | Анестезиолог |
| `room_name` | Операционная |
| `protocol_text` | Текст протокола |
| `complications` | Осложнения |
| `blood_loss_ml` | Кровопотеря (мл) |
| `implants_used` | Использованные импланты |
| `postop_instructions` | Послеоперационные назначения |

## Статусы

| Статус | Описание |
|--------|----------|
| `planned` | Запланирована |
| `in_progress` | В процессе |
| `completed` | Завершена |
| `cancelled` | Отменена |
| `postponed` | Отложена |

## API

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/v1/surgery/` | Создать протокол |
| GET | `/api/v1/surgery/` | Список (фильтры: status, surgeon_id, date_from, date_to) |
| GET | `/api/v1/surgery/{id}` | Детали |
| PATCH | `/api/v1/surgery/{id}` | Обновить |
| PATCH | `/api/v1/surgery/{id}/start` | Начать операцию |
| PATCH | `/api/v1/surgery/{id}/complete` | Завершить |
| PATCH | `/api/v1/surgery/{id}/cancel` | Отменить |
| GET | `/api/v1/surgery/schedule` | Расписание (ближайшие N дней) |
| GET | `/api/v1/surgery/{id}/print` | Печатный HTML протокол |
