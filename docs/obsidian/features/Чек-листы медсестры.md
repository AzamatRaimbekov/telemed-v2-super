# Чек-листы медсестры (Compliance Checklists)

> Система контрольных чек-листов для медсестёр. Шаблоны чек-листов по категориям, заполнение для пациента, отчёт по комплаенсу, контроль просроченных.

---

## Обзор

Категории шаблонов:
- `nursing` — сестринский уход
- `infection` — инфекционный контроль
- `surgery_prep` — подготовка к операции
- `discharge` — выписка

Каждый шаблон содержит список пунктов (items) с флагом `required`. При заполнении автоматически рассчитывается процент выполнения и флаг полного завершения.

---

## Архитектура

### Модели данных

**Таблица:** `checklist_templates`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | UUID | PK |
| `clinic_id` | UUID | Мультитенант |
| `name` | String(200) | Название шаблона |
| `category` | String(50) | Категория |
| `items` | JSON | Пункты чек-листа |
| `description` | Text | Описание |

**Таблица:** `checklist_instances`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | UUID | PK |
| `clinic_id` | UUID | Мультитенант |
| `template_id` | UUID | FK -> checklist_templates |
| `patient_id` | UUID | FK -> patients |
| `completed_by_id` | UUID | FK -> users (медсестра) |
| `checklist_date` | Date | Дата заполнения |
| `responses` | JSON | Ответы по пунктам |
| `is_complete` | Boolean | Все пункты выполнены |
| `completion_percent` | Integer | Процент выполнения |
| `notes` | Text | Примечания |

### Бэкенд

- **Модели:** `backend/app/models/checklist.py`
- **Маршруты:** `backend/app/api/v1/routes/checklists.py`

## API Эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/v1/checklists/templates` | Список шаблонов |
| POST | `/api/v1/checklists/templates` | Создать шаблон |
| PUT | `/api/v1/checklists/templates/{id}` | Обновить шаблон |
| DELETE | `/api/v1/checklists/templates/{id}` | Удалить шаблон (soft) |
| POST | `/api/v1/checklists/fill` | Заполнить чек-лист для пациента |
| GET | `/api/v1/checklists/patient/{patient_id}` | Чек-листы пациента |
| GET | `/api/v1/checklists/{id}` | Детали чек-листа |
| GET | `/api/v1/checklists/compliance-report` | Отчёт комплаенса по медсёстрам |
| GET | `/api/v1/checklists/overdue` | Пациенты без сегодняшних чек-листов |

---

## Бизнес-логика

1. Администратор создаёт шаблоны с пунктами (`items` — JSON массив)
2. Медсестра заполняет чек-лист для пациента (POST `/fill`)
3. Автоматически считается `completion_percent` и `is_complete`
4. Отчёт комплаенса показывает средний % выполнения по каждой медсестре
5. Endpoint `/overdue` выявляет пациентов без сегодняшних чек-листов

### Формат items
```json
[
  {"id": "1", "text": "Измерить температуру", "required": true},
  {"id": "2", "text": "Проверить повязку", "required": true},
  {"id": "3", "text": "Записать жалобы", "required": false}
]
```

### Формат responses
```json
{
  "1": {"checked": true, "note": "36.6"},
  "2": {"checked": true, "note": ""},
  "3": {"checked": false, "note": "Пациент спал"}
}
```
