# Больничный лист (Sick Leave)

> Электронный листок нетрудоспособности. Выдача, продление, закрытие и верификация больничных листов. Печатная HTML-форма, публичная проверка по номеру.

---

## Обзор

Статусы: `draft` → `issued` → `extended` → `closed` / `cancelled`

Автоматическая генерация уникального номера в формате `BL-YYYYMMDDHHMMSS-XXXXXX`.

---

## Архитектура

### Модель данных

**Таблица:** `sick_leaves`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | UUID | PK |
| `clinic_id` | UUID | Мультитенант |
| `patient_id` | UUID | FK -> patients |
| `doctor_id` | UUID | FK -> users |
| `number` | String(30) | Уникальный номер БЛ |
| `diagnosis_code` | String(20) | Код МКБ-10 |
| `diagnosis_name` | String(300) | Название диагноза |
| `start_date` | Date | Дата начала |
| `end_date` | Date | Дата окончания |
| `extension_days` | Integer | Дни продления |
| `status` | Enum | draft/issued/extended/closed/cancelled |
| `employer_name` | String(200) | Название работодателя |
| `notes` | Text | Примечания |

### Бэкенд

- **Модель:** `backend/app/models/sick_leave.py`
- **Маршруты:** `backend/app/api/v1/routes/sick_leave.py`

## API Эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/v1/sick-leave/` | Выдать больничный |

---

## Бизнес-логика

1. Врач создаёт больничный с датами и диагнозом
2. Генерируется уникальный номер
3. Можно продлить — `end_date` сдвигается, `extension_days` суммируется
4. Закрытие фиксирует статус `closed`
5. Печатная форма содержит все данные + подпись врача + печать клиники
6. Публичная верификация по номеру (без авторизации)
