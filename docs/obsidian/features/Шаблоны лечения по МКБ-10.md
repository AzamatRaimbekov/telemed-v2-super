# Шаблоны лечения по МКБ-10 (Treatment Templates)

## Описание

Система шаблонов лечения, привязанных к кодам МКБ-10 (Международная классификация болезней). Врач может создать шаблон для конкретного диагноза с назначениями препаратов, процедур, диетой и рекомендациями. При применении шаблона автоматически создаётся план лечения (TreatmentPlan) с позициями (TreatmentPlanItem).

## Модель БД

### `treatment_templates`

| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID PK | Идентификатор |
| clinic_id | UUID FK | Клиника (тенант) |
| icd10_code | String(20) | Код МКБ-10 (напр. I63.0) |
| icd10_name | String(300) | Название диагноза |
| template_name | String(200) | Название шаблона |
| medications | JSON | Список препаратов |
| procedures | JSON | Список процедур |
| recommendations | Text | Рекомендации |
| diet | String(200) | Диета |
| created_by_id | UUID FK | Автор шаблона |
| is_deleted | Boolean | Мягкое удаление |
| created_at | DateTime | Дата создания |
| updated_at | DateTime | Дата обновления |

### Формат `medications` (JSON)

```json
[
  {"name": "Аспирин", "dosage": "100мг", "frequency": "1 раз/день", "duration": "30 дней"},
  {"name": "Клопидогрел", "dosage": "75мг", "frequency": "1 раз/день", "duration": "90 дней"}
]
```

### Формат `procedures` (JSON)

```json
[
  {"name": "ЭКГ", "frequency": "при поступлении"},
  {"name": "ОАК", "frequency": "ежедневно"},
  {"name": "МРТ головного мозга", "frequency": "контрольное через 7 дней"}
]
```

## API Endpoints

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/api/v1/treatment-templates/` | Список всех шаблонов |
| GET | `/api/v1/treatment-templates/by-icd/{icd10_code}` | Шаблоны по коду МКБ-10 |
| POST | `/api/v1/treatment-templates/` | Создать шаблон |
| PUT | `/api/v1/treatment-templates/{id}` | Обновить шаблон |
| DELETE | `/api/v1/treatment-templates/{id}` | Удалить шаблон (мягко) |
| POST | `/api/v1/treatment-templates/{id}/apply` | Применить шаблон к пациенту |

## Применение шаблона

**POST** `/api/v1/treatment-templates/{id}/apply`

```json
{"patient_id": "uuid"}
```

Создаёт:
- `TreatmentPlan` со статусом DRAFT
- `TreatmentPlanItem` типа MEDICATION для каждого препарата
- `TreatmentPlanItem` типа PROCEDURE для каждой процедуры
- `TreatmentPlanItem` типа DIET если указана диета
