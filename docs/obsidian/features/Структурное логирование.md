# Структурное логирование

> JSON-логи для продакшен мониторинга. Совместимо с Grafana Loki, ELK, Datadog.

---

## Архитектура

**Модуль** (`backend/app/core/structured_logging.py`):
- Использует `structlog` для структурированного вывода
- JSON формат для продакшена, console renderer для разработки
- ISO таймстемпы, уровни логов, имена логгеров
- `RequestLogger` — хелпер для логирования API запросов

---

## Использование

### Инициализация
```python
from app.core.structured_logging import setup_logging

# В lifespan приложения
setup_logging(log_level="INFO", json_format=True)
```

### Логирование
```python
from app.core.structured_logging import get_logger

logger = get_logger("patients")

logger.info("patient_created", patient_id="123", clinic_id="1")
logger.error("validation_failed", field="inn", value="invalid")
```

### RequestLogger
```python
from app.core.structured_logging import request_logger

await request_logger.log_request(
    method="POST",
    path="/api/v1/patients",
    status=201,
    duration_ms=45.2,
    user_id="doc-1",
    clinic_id="clinic-1"
)
```

---

## Формат вывода (JSON)

```json
{
  "event": "api_request",
  "method": "POST",
  "path": "/api/v1/patients",
  "status": 201,
  "duration_ms": 45.2,
  "user_id": "doc-1",
  "timestamp": "2026-04-23T10:15:30.123456Z",
  "level": "info",
  "logger": "api.request"
}
```

---

## Интеграция с Grafana

Логи в JSON формате парсятся Grafana Loki / Promtail без дополнительной настройки. Можно строить дашборды по:
- `status` — распределение HTTP кодов
- `duration_ms` — время ответа (P50, P95, P99)
- `path` — самые популярные эндпоинты
- `level=error` — количество ошибок
