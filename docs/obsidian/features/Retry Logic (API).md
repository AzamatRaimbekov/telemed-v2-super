# Retry Logic для API-запросов

> Автоматический повтор неудачных API-запросов с экспоненциальной задержкой. Повышает устойчивость к временным сетевым ошибкам.

---

## Конфигурация

Настроено в `QueryClient` (`frontend/src/lib/query-client.ts`):

### Queries (GET-запросы)
| Параметр | Значение | Описание |
|----------|---------|----------|
| `retry` | 2 | Макс. 2 повтора при ошибке |
| `retryDelay` | Экспоненциальный | 1с → 2с → 4с (макс 10с) |
| `staleTime` | 30 000 мс | Данные «свежие» 30 сек |
| `gcTime` | 300 000 мс | Кэш живёт 5 минут |
| `refetchOnWindowFocus` | false | Без рефетча при фокусе |

### Mutations (POST/PUT/DELETE)
| Параметр | Значение | Описание |
|----------|---------|----------|
| `retry` | 1 | Один повтор |
| `retryDelay` | 1 000 мс | Фиксированная задержка 1с |

## Формула задержки

```
delay = min(1000 * 2^attemptIndex, 10000)
```

- Попытка 0: 1 секунда
- Попытка 1: 2 секунды
- Попытка 2+: 4 секунды (но не больше 10 сек)
