# Ролевая матрица (управление правами)

UI для управления ролями и правами доступа. Доступна суперадмину и админу клиники.

## Структура прав

### 9 групп пермишенов (~50 прав)

| Группа | Код | Примеры прав |
|--------|-----|-------------|
| Пациенты | `patients:*` | create, read_own, read_all, update, delete, assign_staff |
| Мед. карта | `medical_card:*` | read_own, read_all, write_notes, vitals_record |
| Лечение | `treatment:*` | create_plan, prescribe, order_procedure, order_lab |
| Аптека | `pharmacy:*` | view_stock, dispense, manage_stock, purchase_orders |
| Лаборатория | `lab:*` | view_orders, collect_sample, enter_results, approve_results |
| Расписание | `schedule:*` | view_own, view_all, manage_appointments, manage_shifts |
| Биллинг | `billing:*` | view_invoices, create_invoice, process_payment |
| Персонал | `staff:*` | view, create, edit, deactivate, manage_permissions |
| Отчёты | `reports:*` | view_clinical, view_financial, view_operational, export |

### 7 шаблонов ролей

| Шаблон | Кол-во прав | Описание |
|--------|------------|----------|
| Врач | 18 | Полный доступ к пациентам, лечению, назначениям |
| Медсестра | 8 | Витальные показатели, листы назначений |
| Ресепшн | 8 | Регистрация, расписание, платежи |
| Фармацевт | 4 | Аптека и выдача лекарств |
| Лаборант | 4 | Анализы и результаты |
| Физиотерапевт | 7 | Реабилитация и упражнения |
| Клиник Админ | Все | Полный доступ ко всему |

### Система оверрайдов

Для конкретного сотрудника можно:
- **Дать** дополнительное право (помимо шаблона)
- **Забрать** право из шаблона
- Каждый оверрайд с причиной и аудитом

## UI

### Вкладка "Матрица прав"
- Таблица: шаблоны (колонки) × группы пермишенов (строки)
- Чекбоксы для включения/выключения
- Сворачиваемые секции по группам
- Цветовая кодировка групп
- Индикатор несохранённых изменений
- Сохранение/отмена

### Вкладка "Шаблоны ролей"
- Карточки шаблонов
- Создание, дублирование, удаление

## API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/v1/permissions` | Все группы и права |
| GET | `/api/v1/permission-templates` | Список шаблонов |
| GET | `/api/v1/permission-templates/{id}` | Шаблон с правами |
| POST | `/api/v1/permission-templates` | Создать шаблон |
| PUT | `/api/v1/permission-templates/{id}` | Обновить шаблон |
| DELETE | `/api/v1/permission-templates/{id}` | Удалить шаблон |
| POST | `/api/v1/staff/{id}/permissions/grant` | Дать право сотруднику |
| POST | `/api/v1/staff/{id}/permissions/revoke` | Забрать право |

## Frontend система прав

### RequirePermission компонент

Проверка гранулярных прав (не только роли):

```tsx
<RequirePermission permission="treatment:prescribe">
  <PrescribeButton />
</RequirePermission>

<RequirePermission any={["pharmacy:dispense", "pharmacy:manage_stock"]}>
  <PharmacyPanel />
</RequirePermission>
```

**Файл:** `frontend/src/components/shared/require-permission.tsx`

### usePermission хук

```typescript
const canPrescribe = usePermission("treatment:prescribe");
const canAnyPharmacy = usePermissionAny(["pharmacy:dispense", "pharmacy:manage_stock"]);
const canAllStaff = usePermissionAll(["staff:view", "staff:create"]);
```

**Файл:** `frontend/src/hooks/usePermission.ts`

### Загрузка прав при логине

При `fetchUser()` автоматически загружаются effective permissions через `/staff/{id}/permissions`. Хранятся в `user.permissions: string[]` в auth store.

SUPER_ADMIN и CLINIC_ADMIN автоматически получают `["*"]` (полный доступ).

### Управление правами сотрудника

Страница `/rbac-user?userId=...` — индивидуальное управление правами:
- Все группы с пермишенами
- Показывает что из шаблона, что добавлено/отозвано вручную
- Кнопки Grant/Revoke с причиной
- Цветовая индикация: шаблон (фиолетовый), добавлено (+зелёный), отозвано (−красный)

**Файл:** `frontend/src/routes/_authenticated/rbac-user.tsx`
