# Рассрочка (Installment Plans)

> Оплата лечения в рассрочку: создание плана, график платежей, прогресс оплаты.

---

## Обзор

Клиника создаёт план рассрочки для пациента: общая сумма делится на N ежемесячных платежей. Пациент видит прогресс-бар оплаты и график в личном кабинете. При полной оплате план автоматически закрывается.

---

## Архитектура

### Модели данных

**Таблица:** `installment_plans`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | UUID | PK |
| `clinic_id` | UUID | Мультитенант |
| `patient_id` | UUID | FK -> patients |
| `total_amount` | Float | Общая сумма |
| `paid_amount` | Float | Оплачено |
| `installments_count` | Integer | Кол-во платежей |
| `monthly_payment` | Float | Ежемесячный платёж |
| `start_date` | Date | Дата начала |
| `description` | String(300) | Описание |
| `is_active` | Boolean | Активен |
| `is_completed` | Boolean | Завершён |

**Таблица:** `installment_payments`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | UUID | PK |
| `clinic_id` | UUID | Мультитенант |
| `plan_id` | UUID | FK -> installment_plans |
| `amount` | Float | Сумма платежа |
| `payment_date` | Date | Дата платежа |
| `payment_number` | Integer | Номер платежа |
| `is_paid` | Boolean | Оплачен |

## API Эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/v1/installments/` | Создать план рассрочки |
| GET | `/api/v1/installments/patient/{patient_id}` | Планы пациента |
| POST | `/api/v1/installments/{plan_id}/pay` | Записать платёж |
| GET | `/api/v1/installments/{plan_id}` | Детали плана с графиком |
| GET | `/api/v1/portal/installments/my` | Портал: мои рассрочки |
