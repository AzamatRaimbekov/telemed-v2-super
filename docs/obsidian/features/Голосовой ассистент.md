# Голосовой ассистент

AI-powered голосовое управление порталом пациента.

## Архитектура

```
Микрофон → WebSpeech API → STT → AI (Gemini/DeepSeek)
    → Intent Recognition → Action Execution → TTS → Ответ
```

## Backend

**Путь:** `backend/app/services/voice_assistant/`

| Файл | Описание |
|------|----------|
| `service.py` (11KB) | Оркестрация: приём аудио → AI → ответ |
| `action_executor.py` (4KB) | Выполнение распознанных команд |
| `context_builder.py` (5KB) | Контекст пациента для AI |
| `ai_providers/` | Провайдеры: Gemini, DeepSeek, Whisper |

**API маршрут:** `portal_voice.py` — `POST /api/v1/portal/voice/process`

## Frontend

**Путь:** `frontend/src/features/voice-assistant/`

### Компоненты
- `VoiceAssistantProvider.tsx` — React Context провайдер
- `FloatingMic.tsx` — Плавающая кнопка микрофона
- `SpeechBubble.tsx` — Пузыри с ответами AI
- `ConfirmationDialog.tsx` — Подтверждение действий
- `HintChips.tsx` — Подсказки голосовых команд

### Хуки
- `useAIAssistant.ts` — Основной хук голосового ассистента
- `useWakeWord.ts` — Детекция пробуждающего слова
- `useVoiceRecognition.ts` — Web Speech API (STT)
- `useIntentRouter.ts` — Маршрутизация интентов

### Утилиты
- `intents/` — Определения интентов
- `constants.ts` — Подсказки, команды
- `utils/levenshtein.ts` — Fuzzy matching

## Возможности

- Запись на приём голосом
- Навигация по порталу
- Проверка результатов анализов
- Вопросы о лечении
- Подсказки при бездействии (idle hints)
- Включён по умолчанию

## AI провайдеры

| Провайдер | Назначение |
|-----------|-----------|
| **Whisper** | Speech-to-Text (локальный) |
| **Google Gemini** | NLU — понимание команд |
| **DeepSeek** | Альтернативный NLU |
