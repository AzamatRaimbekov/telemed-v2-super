# Контроль инфекций (Infection Control)

## Описание

Система учёта и контроля инфекционных случаев в клинике. Регистрация инфекций, типы изоляции (контактная, капельная, воздушная, защитная), отслеживание карантинных палат, контактный трейсинг.

## Модели БД

### `infection_records`
| Поле | Тип | Описание |
|------|-----|----------|
| patient_id | UUID FK | Пациент |
| reported_by_id | UUID FK | Кто зарегистрировал |
| infection_type | String(200) | Тип инфекции (COVID-19, MRSA и т.д.) |
| isolation_type | Enum | contact / droplet / airborne / protective |
| status | Enum | suspected / confirmed / resolved / monitoring |
| detected_date | Date | Дата обнаружения |
| resolved_date | Date | Дата разрешения |
| room_id | UUID | Палата (опционально) |
| is_quarantined | Boolean | Карантин активен |
| precautions | Text | Меры предосторожности |
| contact_trace | Text | Контактные лица |
| notes | Text | Примечания |

## API эндпоинты

| Метод | Путь | Описание | Авторизация |
|-------|------|----------|-------------|
| POST | `/infection-control/` | Зарегистрировать инфекцию | Staff |
| GET | `/infection-control/` | Список инфекций (фильтр: status, isolation_type) | Staff |
| GET | `/infection-control/active-count` | Количество активных инфекций по типу | Staff |
| GET | `/infection-control/quarantine-rooms` | Палаты на карантине | Staff |
| GET | `/infection-control/{id}` | Детали записи | Staff |
| PATCH | `/infection-control/{id}` | Обновить запись | Staff |
| PATCH | `/infection-control/{id}/resolve` | Отметить как разрешённую | Staff |
