## Принципы организации

### Backend: Layered Architecture
```
Routes → Services → Repositories → Models
  ↓          ↓            ↓           ↓
Schemas   Business    CRUD ops    SQLAlchemy
(Pydantic) Logic      (base.py)   (PostgreSQL)
```

### Frontend: Feature-based + File Routing
```
routes/           → Страницы (TanStack Router)
features/         → API + логика по доменам
components/ui/    → Переиспользуемые UI компоненты
stores/           → Глобальный стейт (Zustand)
lib/              → Утилиты и API клиенты
```
