# Маршрутизация

**Библиотека:** TanStack Router 1.95 (file-based routing)

## Принцип

Каждый файл в `src/routes/` автоматически становится маршрутом:
- `dashboard.tsx` → `/dashboard`
- `patients.index.tsx` → `/patients`
- `patients.$patientId.tsx` → `/patients/:patientId`
- `patients.$patientId/treatment.tsx` → `/patients/:patientId/treatment`

## Карта маршрутов

```
/                           → Landing page
/login                      → Вход для персонала

/_authenticated/            → Layout (sidebar, header)
  /dashboard                → Главная панель
  /patients                 → Список пациентов
  /patients/new             → Регистрация пациента
  /patients/:id             → Карточка пациента
    /medical-card           → Медкарта
    /history                → История
    /treatment              → Лечение
    /labs                   → Анализы
    /procedures             → Процедуры
    /medications            → Лекарства
    /stroke                 → Реабилитация
    /documents              → Документы
    /billing                → Счета
  /staff                    → Список персонала
  /staff/new                → Новый сотрудник
  /staff/:id                → Профиль сотрудника
  /schedule                 → Расписание
  /telemedicine             → Видеоконсультации
  /laboratory               → Лаборатория
  /medicine-settings        → Аптека
  /finance                  → Финансы
  /analytics                → Аналитика
  /audit                    → Аудит
  /notifications            → Уведомления
  /infrastructure           → BMS
    /buildings              → Здания
    /sensors                → Датчики
    /equipment              → Оборудование
    /alerts                 → Алерты
    /automation             → Автоматизация
    /analytics              → BMS аналитика
    /map                    → Карта здания
    /maintenance            → Обслуживание

/portal/                    → Портал пациента
  /login                    → Вход по телефону
  /_portal/                 → Layout портала
    /dashboard              → Главная
    /medical-card           → Медкарта
    /history                → История
    /results                → Результаты анализов
    /appointments           → Запись к врачу
    /exercises              → Упражнения
    /exercise-session       → Сессия (MediaPipe)
    /exercise-progress      → Прогресс
    /recovery               → Восстановление
    /treatment              → Лечение
    /messages               → Сообщения
    /billing                → Счета
    /profile                → Профиль
    /schedule               → Расписание
```

## Защита маршрутов

- `_authenticated.tsx` — проверяет JWT в auth-store
- `_portal.tsx` — проверяет JWT портала пациента
- `<RequireRole>` — компонент для ограничения по роли
