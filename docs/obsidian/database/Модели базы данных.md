# Модели базы данных

Все модели в `backend/app/models/`. ORM: SQLAlchemy 2.0 (async). БД: PostgreSQL 16.

## Базовые классы

**`base.py`:**
- `Base` — declarative base
- `TenantMixin` — `clinic_id` для мультитенантности
- `BaseMixin` — `id`, `created_at`, `updated_at`

## Схема моделей

### Управление доступом

```
User ──┬── Role ──── Permission
       │              │
       └── RBAC ──── PermissionGroup
                      PermissionTemplate
                      UserTemplate
```

- **User** — email, hashed_password, is_active, role
- **Role** — name (admin, doctor, nurse, pharmacist, lab_tech, radiologist, billing...)
- **Permission** — resource + action granular access
- **PermissionGroup** — группировка прав
- **PermissionTemplate** — шаблоны ролей
- **UserTemplate** — привязка user → template

### Пациенты и медицина

```
Clinic ──── Patient ──┬── MedicalCard ──── Visit
                      ├── PatientGuardian
                      ├── VitalSign
                      ├── Diagnosis (ICD-10)
                      ├── MedicalHistoryEntry
                      ├── Document
                      └── FaceSnapshot ── FaceEmbedding
```

- **Patient** — ФИО, дата рождения, ИНН, адрес, телефон, страховка
- **MedicalCard** — master record пациента
- **Visit** — приём у врача (жалобы, осмотр, заключение)
- **VitalSign** — АД, пульс, SpO2, температура
- **Diagnosis** — код МКБ-10 + описание
- **MedicalHistoryEntry** — хронология событий
- **Document** — загруженные файлы (PDF, изображения)
- **FaceSnapshot/FaceEmbedding** — биометрия для регистрации

### Лечение и реабилитация

```
TreatmentPlan ──── TreatmentPlanItem
                          │
StrokeAssessment ──── RehabGoal ──── RehabProgress
                          │
RecoveryGoal ──── RecoveryDomainWeight
                          │
Exercise ──── ExerciseSession ──── ExerciseRep
```

- **TreatmentPlan** — план лечения (статус, даты)
- **TreatmentPlanItem** — конкретное назначение
- **StrokeAssessment** — оценка NIHSS, mRS
- **RehabGoal** — цель реабилитации
- **RehabProgress** — прогресс к цели
- **RecoveryGoal** — цели восстановления с весами
- **Exercise** — упражнение (MediaPipe)
- **ExerciseSession** — сессия упражнений
- **ExerciseRep** — повторение (углы, форма)

### Фармация

```
Drug ──── Prescription
           │
Medication (пациент)
           │
Supplier ──── PurchaseOrder
```

- **Drug** — лекарство (название, форма, дозировка)
- **Prescription** — рецепт врача
- **Medication** — назначение пациенту
- **Supplier** — поставщик
- **PurchaseOrder** — заказ поставщику

### Лаборатория

```
LabTestCatalog ──── LabOrder ──── LabResult
```

- **LabTestCatalog** — каталог доступных анализов
- **LabOrder** — заказ на анализ
- **LabResult** — результат анализа

**Типы датчиков:** HEART_RATE, SPO2, BODY_TEMPERATURE, BLOOD_PRESSURE, FALL_DETECTOR, RESPIRATORY_RATE, ECG, GLUCOSE, WEIGHT, ACTIVITY

### BMS (Управление зданием)

```
Building ──── Floor ──── Zone ──── BmsRoom
BmsSensor ──── BmsSensorReading

```

**Типы помещений:** WARD, CORRIDOR, OPERATING, LAB, PHARMACY, RECEPTION, OFFICE, SERVER_ROOM, STORAGE, ELEVATOR, STAIRCASE, PARKING

**Типы оборудования:** AC, HEATER, LIGHT, VENTILATION, DOOR_LOCK, ELEVATOR, FIRE_SUPPRESSION, GENERATOR, UPS, WATER_PUMP

**Типы BMS датчиков:** TEMPERATURE, HUMIDITY, CO2, LIGHT, SMOKE, WATER_LEAK, MOTION, DOOR, POWER_METER, AIR_QUALITY
