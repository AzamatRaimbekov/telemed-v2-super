# Тесты (Unit + Integration)

## Стек тестирования

- **pytest** >= 8.0 — фреймворк для тестов
- **pytest-asyncio** >= 0.23 — поддержка async тестов
- **httpx** — async HTTP клиент для интеграционных тестов

## Запуск тестов

```bash
cd backend
pytest tests/ -v
```

Только конкретный файл:
```bash
pytest tests/test_health.py -v
```

С покрытием (если установлен pytest-cov):
```bash
pytest tests/ --cov=app --cov-report=term-missing
```

## Что покрыто тестами

| Тест | Файл | Что проверяет |
|------|-------|---------------|
| Health endpoint | `test_health.py` | GET `/api/v1/health` возвращает `{"status": "ok"}` |
| QR генерация | `test_qr.py` | Генерация PNG QR-кода пациента |
| Браслет UID | `test_wristband.py` | Формат `MC-XXXXXX`, уникальность |
| Excel экспорт | `test_export.py` | Генерация XLSX файла (magic bytes `PK`) |
| AI суммаризация | `test_ai_summarizer.py` | Пустая суммаризация (fallback без API) |

## Структура

```
backend/
├── pytest.ini          # Конфигурация pytest
└── tests/
    ├── __init__.py
    ├── conftest.py         # Фикстуры (event_loop)
    ├── test_health.py      # Интеграционный тест API
    ├── test_qr.py          # Unit тест QR генератора
    ├── test_wristband.py   # Unit тест браслетов
    ├── test_export.py      # Unit тест Excel экспорта
    └── test_ai_summarizer.py # Unit тест AI суммаризации
```

## Конфигурация (`pytest.ini`)

```ini
[pytest]
asyncio_mode = auto
testpaths = tests
python_files = test_*.py
python_functions = test_*
```

- `asyncio_mode = auto` — автоматическое определение async тестов
- `testpaths = tests` — папка с тестами

## Добавление нового теста

1. Создайте файл `tests/test_<module>.py`
2. Добавьте функции с префиксом `test_`
3. Для async тестов используйте `@pytest.mark.asyncio`
4. Запустите `pytest tests/ -v`
