# Работа с миграциями

**ORM:** SQLAlchemy 2.0 (async)
**Инструмент:** Alembic 1.14

## Расположение

```
backend/alembic/
├── env.py              # Конфигурация Alembic
├── versions/           # 17 файлов миграций
└── alembic.ini         # (в корне backend)
```

## Команды

```bash
cd backend

# Создание миграции (автогенерация из моделей)
alembic revision --autogenerate -m "описание изменений"

# Применение всех миграций
alembic upgrade head

# Откат на 1 миграцию
alembic downgrade -1

# Просмотр текущей ревизии
alembic current

# История миграций
alembic history
```

## Автоматическое применение

В продакшене миграции применяются автоматически при старте:
`backend/app/main.py` → lifespan → `alembic upgrade head`

## Правила

1. Всегда создавай миграцию после изменения модели
2. Проверь сгенерированный SQL перед применением
3. Миграции должны быть обратимыми (upgrade + downgrade)
4. Не редактируй уже применённые миграции
5. Seed скрипты идемпотентны — не зависят от порядка миграций
