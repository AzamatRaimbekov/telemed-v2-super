from __future__ import annotations

    status: str

### Task 3: Backend — Pharmacy Service

**Files:**
- Create: `backend/app/services/pharmacy.py`

- [ ] **Step 1: Create PharmacyService with all methods**

The service class takes `AsyncSession` in __init__. Implement these methods:

**Dashboard:**
- `get_dashboard(clinic_id)` → dict with stats (drug_count, stock_value, pending_prescriptions, expired_batches, low_stock_count), alerts list, recent_operations (last 10 InventoryLog entries)

**Inventory:**
- `list_inventory(clinic_id, search, category, form, status, skip, limit)` → aggregated drug list with total_quantity, nearest_expiry, status per drug. Query Inventory grouped by drug_id, join Drug for name/form/category. Status logic: "expired" if any batch expired, "out" if total_qty=0, "low" if total_qty <= threshold, else "ok".
- `get_batches(clinic_id, drug_id)` → list of Inventory records for specific drug, ordered by expiry_date ASC
- `write_off(clinic_id, inventory_id, quantity, reason, user_id)` → decrease batch quantity, create InventoryLog(WRITE_OFF)
- `adjust(clinic_id, inventory_id, new_quantity, reason, user_id)` → set batch quantity, create InventoryLog(ADJUSTMENT) with quantity_change = new - old

**Dispensing:**
- `get_prescription_queue(clinic_id, search, date_filter, skip, limit)` → Prescriptions with status ACTIVE, join patient/doctor names, count items
- `get_prescription_detail(clinic_id, prescription_id)` → Prescription with items, each item enriched with available_stock (sum of Inventory.quantity for that drug) and suggested_batch (FIFO — batch with nearest expiry that has quantity > 0)
- `dispense(clinic_id, prescription_id, items: list[DispenseItemRequest], user_id)` → For each item: find inventory batch (FIFO if not specified), decrease quantity, create DispenseRecord, create InventoryLog(DISPENSE). Then check if all items dispensed → set prescription status to DISPENSED or PARTIALLY_DISPENSED.

**Purchase Orders:**
- `list_orders(clinic_id, status, supplier_id, skip, limit)` → PurchaseOrder list with order_items count, total_amount
- `get_order(clinic_id, order_id)` → PurchaseOrder with order_items (including drug details)
- `create_order(clinic_id, data: PurchaseOrderCreate, user_id)` → Create PurchaseOrder + PurchaseOrderItems, calc total_amount
- `update_order(clinic_id, order_id, data: PurchaseOrderUpdate)` → Update draft order only
- `submit_order(clinic_id, order_id)` → Set status SUBMITTED, set ordered_at
- `receive_order(clinic_id, order_id, data: ReceiveOrderRequest, user_id)` → For each item: create Inventory batch, create InventoryLog(RECEIPT), set quantity_received. Set order status RECEIVED, received_at.
- `cancel_order(clinic_id, order_id)` → Set status CANCELLED (only DRAFT/SUBMITTED)

**Suppliers:**
- `list_suppliers(clinic_id, search, skip, limit)` → Supplier list with order count
- `create_supplier(clinic_id, data)` → Create Supplier
- `update_supplier(clinic_id, supplier_id, data)` → Update Supplier
- `delete_supplier(clinic_id, supplier_id)` → Soft delete if no orders, else raise error

All methods follow existing patterns from `backend/app/services/settings.py` — query with filters, `.scalars().all()`, use `func.count()` for totals.
